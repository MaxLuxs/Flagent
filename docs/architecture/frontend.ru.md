# Архитектура Frontend

> [English](frontend.md) | Русский

## Обзор

Admin UI Flagent собран на **Compose for Web** (Kotlin/JS): одностраничное приложение для управления feature flags, сегментами, экспериментами и аналитикой. Используется общая дизайн-система и навигация с сайтом документации.

## Стек

- **UI:** Compose for Web (Kotlin/JS)
- **HTTP:** Ktor client (или platform fetch)
- **State:** состояние компонентов и опционально общее состояние
- **Сборка:** Gradle (JS target)

## Основные компоненты

1. **Dashboard** — краткая статистика, недавние флаги, быстрые действия (создать флаг, консоль evaluation).
2. **Flags List** — список флагов с фильтрами (key, status, tags) и поиском; точка входа для создания/редактирования.
3. **Flag Editor** — создание/редактирование флага: key, описание, статус, правила таргетинга, варианты, распределения.
4. **Segment Editor** — создание/редактирование сегментов и ограничений (region, tier, кастомные атрибуты).
5. **Experiments** — список экспериментов, метрики по вариантам, настройка распределений.
6. **Evaluation / Debug Console** — одиночный и batch evaluation с entity ID и контекстом.
7. **Analytics** — метрики во времени; отчётность по крашам при включении.

## Интеграция с API

Frontend использует REST API backend: `GET/POST/PUT /api/v1/flags`, `POST /api/v1/evaluation`, endpoints для сегментов, экспериментов и аналитики. Запросы идут с того же origin (или настроенного base URL) с аутентификацией (JWT, cookie) при включённой защите.
